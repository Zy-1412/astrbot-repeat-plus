## 快速开始

### 安装

```
1. 下载 astrbot_plugin_repeat_plus_v1.2.2.zip（见 Release）
2. AstrBot WebUI → 插件管理 → 上传插件包
3. 启用插件
4. 群内发送 /复读开启
```

### 依赖

| 依赖 | 版本 | 用途 |
|------|------|------|
| AstrBot | `>=4.0.0` | 运行平台 |
| Pillow | 任意 | 关系图生成（可选，缺失则无法生成关系图） |

```bash
pip install Pillow
```

## 配置详解

插件提供 **9 组 43 项**配置，在 AstrBot WebUI 管理面板中直接调整。以下是完整说明。

### 🎯 复读判定（7 项）

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 触发阈值 | int | 3 | 窗口中多少条相似消息触发复读，越小越灵敏 |
| 窗口大小 | int | 5 | 回顾最近 N 条消息，越大越宽松 |
| 模糊匹配灵敏度 | float | 0.9 | 1.0=完全一致，0.9=允许 10% 差异 |
| 权重衰减 | bool | 关 | 开启后新消息权重更高 |
| 同用户连续刷 | bool | 开 | 关=必须是不同人发相同内容才触发 |
| 最小消息长度 | int | 1 | 低于此长度不参与复读，2 可过滤"啊""？" |
| 最大消息长度 | int | 200 | 超过此长度不参与复读，防长文转发刷屏 |

### ⏱️ 冷却与抑制（4 项）

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 复读冷却 | int | 10s | 同一群两次复读最小间隔 |
| 冷却递增 | bool | 开 | 连续触发时冷却逐次递增（最多 5 倍），5 分钟无触发回落 |
| 重复抑制 | int | 60s | 同一条消息 N 秒内只复读一次 |
| 打断惩罚倍率 | float | 2.0 | 打断对群聊干扰更大，额外延长冷却 |

### 🎭 打断系统（6 项）

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 打断基础概率 | float | 0.1 | 触发复读时进入打断分支的概率，随重复次数递增 |
| 原话洗牌 | bool | 开 | 将消息文字随机打乱后发出 |
| 反向复读 | bool | 开 | 将消息文字反转输出 |
| 自定义话术 | bool | 开 | 从自定义话术列表中随机抽取一句发送 |
| 终止复读 | bool | 关 | 不发送任何消息，仅施加冷却惩罚 |
| 自定义打断话术 | text | 见默认 | 每行一句，启用自定义话术时随机抽取 |

### 🎨 复读风格（2 项）

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 模拟延迟 | string | 0.5-2.0s | 复读前随机等待范围，让机器人更像真人 |
| 极速模式 | bool | 关 | 跳过所有延迟，立即回复 |

### 🛡️ 过滤（3 项）

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 内容黑名单 | regex | `admin\|【` | 匹配此正则的消息跳过复读 |
| 屏蔽群号 | text | 空 | 逗号或换行分隔，也可用 `/复读关闭` 临时关闭 |
| 屏蔽用户 QQ | text | 空 | 这些用户的消息不触发复读 |

### 💕 抽取设置（9 项）

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 开启老公模式 | bool | 开 | `/今日老公` `/抽老公` `/强娶老公` 生效 |
| 开启老婆模式 | bool | 开 | `/今日老婆` `/抽老婆` `/强娶老婆` 生效 |
| 每日抽取次数 | int | 1 | 每人每天最多抽取次数 |
| 活跃天数 | int | 30 | 最近 N 天发过言的人才能进入抽取池 |
| 仅活跃成员可被抽 | bool | 开 | 关=全群成员均可被抽（调 API） |
| 排除用户 QQ | text | 空 | 这些 QQ 号永远不会被抽到 |
| 候选池上限 | int | 500 | 活跃用户超过此数时随机截取 |
| 群组白名单 | text | 空 | 仅这些群启用抽人功能 |
| 群组黑名单 | text | 空 | 这些群禁用抽人功能 |

### 💍 强娶与求婚（5 项）

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 强娶冷却天数 | int | 3 | 使用 `/强娶` 后需等待的天数 |
| 强娶每日次数 | int | 1 | 每人每天强娶次数上限，0=不限制 |
| 求婚每日次数 | int | 3 | 每人每天求婚次数上限，0=不限制 |
| 求婚冷却 | int | 86400s | 两次求婚最小间隔，默认 1 天 |
| 允许强娶机器人 | bool | 关 | 开启后允许强娶机器人 |

### ⚙️ 高级选项（6 项）

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 关键词触发 | bool | 关 | 免 `/` 前缀，直接发"抽老公"即可触发 |

| 双向绑定 | bool | 关 | A 抽到 B 时自动给 B 也分配 A |
| 自动撤回 | bool | 关 | 抽取结果在指定秒数后自动撤回 |
| 撤回延迟 | int | 30s | 自动撤回前等待的秒数 |

## 关系图技术说明

### 布局算法

采用 **Fruchterman-Reingold 力导向布局**，全参数自适应 2~50+ 人：

| 参数 | 自适应策略 |
|------|-----------|
| 节点大小 | 6 档：10 人 32-52px → 50 人 15-23px |
| 初始半径 | 根据节点总数和平均直径计算，确保初始圆环不重叠 |
| 迭代次数 | `n × 5`，50 人 = 250 轮 |
| 向心力 | `0.005 + n × 0.00015`，人数越多引力越强 |
| 温控退火 | 从 `chart_w/4` 线性退火至 `max(1.9, chart_w/(n×6))` |
| 碰撞消解 | `n/2` 轮，gap=10px，确保零重叠 |

### 头像下载

并发 `ThreadPoolExecutor`（最多 16 线程），50 人头像下载从 150 秒降至 ~3 秒。

### 画布规格

- 尺寸：1200×800 px
- 主图区：950×800 px，右侧统计面板 250px
- 贝塞尔曲线连线，箭头指示方向，双向边反向弯曲
- 节点颜色：蓝色=抽取者，红色=被抽者，紫色=双向

## 常见问题

<details>
<summary><b>Q: 关系图只显示 3 个人头？</b></summary>

v1.2.2 已解决。根因是节点半径过大（38-62px），力导向排斥力将节点推到边缘。现在节点大小自适应人数，23 人降至 22-34px，50 人降至 15-23px，配合自适应初始半径和增强向心力，确保全部节点可见。
</details>

<details>
<summary><b>Q: 如何同时开启老公和老婆模式？</b></summary>

管理面板中同时勾选「开启老公模式」和「开启老婆模式」即可。两个模式共享数据池，仅话术和指令前缀不同。用户可用 `/今日老公` 和 `/今日老婆` 分别抽取。
</details>

<details>
<summary><b>Q: 复读太频繁 / 太少怎么办？</b></summary>

调整「触发阈值」（默认 3）和「窗口大小」（默认 5）。阈值越低、窗口越大，越容易触发。冷却时间（默认 10s）和冷却递增也会影响频率。
</details>

<details>
<summary><b>Q: 如何让复读更自然？</b></summary>

- 开启「模拟延迟」（默认 0.5-2.0s），机器人不会秒回
- 关闭「极速模式」
- 关闭「同用户连续刷」，只有不同人发相同内容才触发
- 开启「权重衰减」，新消息权重更高
</details>

<details>
<summary><b>Q: 数据会丢失吗？</b></summary>

不会。排行榜、抽取记录、活跃数据、强娶 CD 全部持久化到 `data/` 目录，跨重启保留。排行榜保留 30 天后自动清理。
</details>

<details>
<summary><b>Q: 如何排除某个群或某个用户？</b></summary>

- 群：在管理面板「屏蔽群号」中填写，或在群内发送 `/复读关闭`
- 用户：在管理面板「屏蔽用户 QQ」中填写
- 抽取池排除：在「排除用户 QQ」中填写，该用户永远不会被抽到
</details>
